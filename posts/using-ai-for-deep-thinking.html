Using AI for Deep Thinking - Not Just Doing

There are two ways to use AI.

You can use it to do things. Or you can use it to think about things.

Most people only use it for the first. They're missing the real power.

The Execution Model

When I need to build something, I use Antigravity (Google's AI coding assistant).

It's fast. It's precise. It writes code, debugs issues, and deploys changes in seconds.

This is the "doing" layer. It's transactional. I tell it what to build, and it builds it.

But before I can tell it what to build, I need to figure out why to build it.

That's where Claude comes in.

The Thinking Model

When I need to figure out something, I use Claude.

Not for code. For strategy.

Here's an example from this blog:

Me: "Should I add an email subscription feature, or is that too pushy for a personal blog?"

Claude: Breaks down the tradeoffs:

  • Email gives you ownership of the audience
  • But it adds friction and expectation
  • Consider: Are you writing to build an audience, or to think out loud?
  • If the former, email is essential. If the latter, RSS is better.

This isn't code. This is thinking.

Claude doesn't just answer the question. It helps me ask better questions.

Real Example: The Collections System

I wanted readers to browse posts by topic. My first instinct was: "Add tags."

But before jumping into implementation, I asked Claude:

Me: "What's the best way to organize 6-10 essays for easy browsing?"

Claude's response (summarized):

  • Tags work, but they fragment the experience if you have too many
  • Categories are simpler but less flexible
  • Hybrid approach: Use categories as primary navigation, tags as secondary filters
  • Consider: Do readers want to "explore by theme" or "find a specific topic"?

This helped me realize I didn't just need tags. I needed a Collections page with a grid layout where readers could see all topics at a glance.

Then I went to Antigravity and said: "Build a Collections page with a grid of tags, each showing post count."

Antigravity built it in 2 minutes.

But the thinking that led to that decision? That took 10 minutes with Claude.

The Division of Labor

Here's how I use both:

Claude (Thinking):

  • "What features should I prioritize?"
  • "How should the navigation flow work?"
  • "What's the user mental model for this?"
  • "Is this feature aligned with my goals?"

Antigravity (Execution):

  • "Add a sorting feature to the homepage"
  • "Fix the date formatting bug"
  • "Deploy the new Collections page"
  • "Update the CSS for the glassmorphism header"

Claude helps me decide what to build. Antigravity helps me build it fast.

Why Claude for Strategy?

Claude has a different personality than ChatGPT or Antigravity.

It's more Socratic. It asks follow-up questions. It challenges assumptions.

When I say: "I want to add dark mode," ChatGPT says: "Here's how to implement it."

Claude says: "Why? Who's your audience? Are they developers who expect dark mode, or general readers who might find it confusing?"

It makes me justify my decisions before I make them.

This saves me from building features I don't actually need.

The Meta Layer

The real unlock is using AI to think about how to think.

Example: I was stuck on whether to make this blog personal or professional.

Me to Claude: "Should I write in a polished, editorial voice or a casual, conversational one?"

Claude: "What's the purpose of the blog? If it's to build authority, polish matters. If it's to develop ideas in public, authenticity matters more. What do you want readers to take away?"

This forced me to clarify: I'm not building a media brand. I'm thinking out loud.

That realization shaped everything: the tone, the design, the lack of SEO optimization.

Without Claude, I would have built a blog that looked professional but felt wrong.

The Workflow

Here's my actual process:

  1. Idea Phase: Talk to Claude about what I'm trying to achieve
  2. Strategy Phase: Claude helps me break down the problem and weigh tradeoffs
  3. Execution Phase: Antigravity builds what we decided on
  4. Iteration Phase: If something feels off, back to Claude to rethink the approach

This loop is fast but thoughtful.

I'm not just shipping features. I'm shipping the right features.

The Difference

Most people treat AI like a tool: "Do this task for me."

That works for execution. But it doesn't work for strategy.

Strategy requires conversation. It requires being challenged. It requires saying "I don't know" and working through it.

Claude is designed for that. It doesn't just give answers. It thinks with you.

Try It

Next time you're building something, try this:

  1. Before you start coding, spend 10 minutes with Claude asking "why"
  2. Ask it to challenge your assumptions
  3. Let it ask you questions instead of just giving answers
  4. Then‚Äîand only then‚Äîswitch to your execution tool and build

You'll be surprised how much clearer your decisions become.

And you'll waste a lot less time building things you didn't actually need.

Because the best code is the code you don't have to write.
